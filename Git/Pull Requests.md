Imagine a central (remote) [[GitHub Repository]] repo with a team contributing code to it. Each one [[Commit]] their work locally and pushes it directly to the GitHub repo's `main` branch. If it sounds like an easy way to have a broken mess of a codebase, it's because it is!

This is why developers merge branches on GitHub through [pull requests](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/about-pull-requests).

On GitHub, a pull request is a way to propose changes, typically to the rest of your team, or to the maintainer of a project you're contributing to.

Pull requests allow team members to see what changes are being proposed and to discuss them before they are merged into the main codebase.
